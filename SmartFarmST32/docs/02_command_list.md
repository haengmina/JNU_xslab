# 일반 제어 명령
이 문서에서는 `센서/제어 노드`에서 **지원하는 일반 명령에 관해** 다룹니다

@important **[Protocol - Packet Format](docs/01_protocol.md#packet-format) 규격을 준수해야합니다**

## Node Select
**통합제어기에서 Transaction을 시작하기 위해,** node 선택하는 용도로 사용됩니다

### Requst
`Packet Format에 따라,` 다음과 같이 설정하여 **노드를 선택할 수 있습니다**
- `Command:` 0x20 (NODE_SELECT_REQUEST)
- `Data Length:` 8
- `Data:` 다음 규격을 따릅니다
```
/*!
 * |  7  |  6  |  5  |  4  |  3  |  2  |  1  |  0  |
 * +-----+-----+-----+-----+-----+-----+-----+-----+----
 * |               Serial Number LSB               | [0]
 * |                                               |
 * |                 Serial Number                 | [1]
 * |                                               |
 * |                 Serial Number                 | [2]
 * |                                               |
 * |                 Serial Number                 | [3]
 * |                                               |
 * |                 Serial Number                 | [4]
 * |                                               |
 * |                 Serial Number                 | [5]
 * |                                               |
 * |                 Serial Number                 | [6]
 * |                                               |
 * |               Serial Number MSB               | [7]
 * +-----+-----+-----+-----+-----+-----+-----+-----+----
 */
```

### Response
- `Command:` 0x21 (NODE_SELECT_RESPONSE)
- `Data Length:` 0

## Accelerometer Read
**3축 가속도 센서의 값을** 읽습니다

```
 * |                                               | [02]

 * |                                               | [03]

 * |                                               | [04]
 * |                                               |

 * +-----+-----+-----+-----+-----+-----+-----+-----+-----
 * Roll Value Type: float (IEEE 754), Unit: degree
 * Pitch Value Type: float (IEEE 754), Unit: degree
 */
```

## 제어 노드 전용 명령
**제어 노드에서만 사용할 수 있는** 명령입니다

@warning **센서 노드를 대상으로 전송하는 경우 `ERROR_INVALID_REQUEST (0x22)` Response가** 전송됩니다

@note **값이 유효하지 않은 Reqeust를 전송하는 경우 `REQUEST_FAILED (0x23)` Response가** 전송됩니다

### Digital Input Read
**지정된 DI 채널의 상태를** 읽습니다

#### Request
- `Command:` 0x30 (DI_READ)
- `Data Length:` 1
- `Data:` 다음 규격을 따릅니다
```
/*!
 * |  7  |  6  |  5  |  4  |  3  |  2  |  1  |  0  |
 * +-----+-----+-----+-----+-----+-----+-----+-----+----
 * |                Channel (1 ~ 4)                | [0]
 * +-----+-----+-----+-----+-----+-----+-----+-----+----
 */
```

#### Response
- `Command:` 0x40 (DI_RESPONSE)
- `Data Length:` 1
- `Data:` 다음 규격을 따릅니다
```
/*!
 * |  7  |  6  |  5  |  4  |  3  |  2  |  1  |  0  |
 * +-----+-----+-----+-----+-----+-----+-----+-----+----
 * |             Input Value (0 or 1)              | [0]
 * +-----+-----+-----+-----+-----+-----+-----+-----+----
 */
```

### Serial Write
**지정된 채널에 데이터를 쓰고, 지정된 시간 동안 수신된 데이터를** 읽습니다

@note 대상 채널이 DDI로 설정된 경우 `Request 시작 시점 부터 종료 시점까지,` **DB9의 +12V 출력이 활성화됩니다**

#### Request
- `Command:` 0x80 (SERIAL_WRITE)
- `Data Length:` 3 + Data Length
- `Data:` 다음 규격을 따릅니다
```
/*!
 * |  7  |  6  |  5  |  4  |  3  |  2  |  1  |  0  |
 * +-----+-----+-----+-----+-----+-----+-----+-----+----
 * |                Channel (1 ~ 2)                | [0]
 * +-----+-----+-----+-----+-----+-----+-----+-----+----
 * |              Receive Timeout LSB              | [1]
 * |                                               |
 * |              Receive Timeout MSB              | [2]
 * +-----+-----+-----+-----+-----+-----+-----+-----+----
 * |                    Data 0                     | [3]
 * ~                                               ~
 * |                    Data N                     | [N]
 * +-----+-----+-----+-----+-----+-----+-----+-----+----
 * Receive Timeout: uint16, Unit: msec
 */
```

@note **전송할 `Data`가 없는 경우,** 0 ~ 2 까지만 Reqeust로 전송할 수 있습니다\n
`Receive Timeout`으로 **지정된 시간 이후에 Response가 전송됩니다**

#### Response
- `Command:` 0x81 (SERIAL_WRITE_RESPONSE)
- `Data Length:` Received Data Length
- `Data:` 다음 규격을 따릅니다
```
/*!
 * |  7  |  6  |  5  |  4  |  3  |  2  |  1  |  0  |
 * +-----+-----+-----+-----+-----+-----+-----+-----+----
 * |                    Data 0                     | [0]
 * ~                                               ~
 * |                    Data N                     | [N]
 * +-----+-----+-----+-----+-----+-----+-----+-----+----
 */
```
